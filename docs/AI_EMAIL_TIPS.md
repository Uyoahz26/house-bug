# AI 邮件文案功能

## 概述

HomeBug 的定时提醒邮件现已集成 AI 智能文案生成功能！系统会为每个需要提醒的物品生成幽默、灵动、个性化的提示文案，让库存管理变得更有趣。

## 功能特点

### 🎭 幽默风趣

AI 会根据物品特性和状态生成有趣的文案，例如：

- **洗发水快过期**："现在可以一天洗三次头了，头皮SPA走起！"
- **牛奶库存不足**："早餐没奶喝，只能干啃面包了🥖"
- **酱油已过期**："这瓶酱油见证了你从单身到现在，该说再见了"

### 🎯 个性化定制

- 每个物品都有独特的文案
- 结合物品名称、品牌、过期天数、库存数量等信息
- 根据不同状态（已过期/即将过期/库存不足）生成不同风格的提示

### 🚀 智能批量处理

- 自动批量生成文案（每批最多 10 个物品）
- 异步并行处理，不影响邮件发送速度
- 优雅降级：AI 不可用时使用默认文案

### 💬 语言风格

- 轻松活泼，像朋友聊天
- 可以使用网络流行语、谐音梗、夸张手法
- 每条文案 15-30 字，简短有力

## 工作原理

### 1. 物品分类

系统将需要提醒的物品分为三类：

- **已过期物品**：调侃但提醒丢弃
- **即将过期物品**：鼓励尽快使用
- **库存不足物品**：提醒补货

### 2. AI 文案生成

```typescript
// 为每类物品批量生成文案
const [expiredTips, expiringTips, lowStockTips] = await Promise.all([
  generateAiTipsForItems(expiredItems, "expired"),
  generateAiTipsForItems(expiringItems, "expiring"),
  generateAiTipsForItems(lowStockItems, "lowStock"),
]);
```

### 3. 文案集成

生成的文案会自动添加到邮件的文本和 HTML 版本中：

**文本版本**：

```
- 海飞丝洗发水（剩余 3 天，库存 1瓶）💡 现在可以一天洗三次头了，头皮SPA走起！
```

**HTML 版本**：

```html
<div
  style="font-size: 13px; color: #6366f1; margin-top: 6px; font-style: italic;"
>
  💡 现在可以一天洗三次头了，头皮SPA走起！
</div>
```

## 配置要求

### 必需条件

1. **启用 AI 功能**：在系统配置中开启 AI
2. **配置 AI 提供商**：支持 OpenAI、Anthropic、豆包等
3. **有效的 API Key**：确保 API Key 有效且有余额

### 推荐配置

- **提供商**：OpenAI（识别准确度高）
- **模型**：gpt-4o-mini（性价比高）
- **温度**：0.7-1.0（更有创意，默认 0.1 可能太保守）

💡 **提示**：如果想要更有创意的文案，可以在系统配置中将 AI 温度参数调高到 0.7-1.0。

## 成本估算

### 单次邮件成本

假设有 20 个需要提醒的物品：

- 输入 tokens: ~800（提示词 + 物品信息）
- 输出 tokens: ~400（20 条文案）
- 单次成本: $0.15 × 0.8/1000 + $0.60 × 0.4/1000 = **$0.00036**（约 ¥0.0026，0.26 分）

### 月度成本

假设每天发送一次提醒邮件：

- 每月成本: $0.00036 × 30 = **$0.0108**（约 ¥0.078，不到 1 毛钱）

非常经济实惠！🎉

## 降级策略

如果 AI 不可用（未配置、请求失败、超时等），系统会自动降级：

- ✅ 邮件正常发送
- ✅ 使用默认的静态文案
- ✅ 不影响用户体验
- ✅ 错误日志记录到控制台

## 技术实现

### 核心函数

#### `generateAiTipsForItems()`

```typescript
async function generateAiTipsForItems(
  items: ReminderItemView[],
  type: "expired" | "expiring" | "lowStock",
): Promise<Map<string, string>>;
```

**功能**：

- 批量生成 AI 文案（每批最多 10 个物品）
- 返回物品 key 到文案的映射
- 失败时返回空 Map（优雅降级）

**提示词设计**：

- 明确角色定位："幽默风趣的家庭管家助手囤囤鼠"
- 具体要求：15-30字、幽默抽象、结合场景
- 风格示例：提供多个示例引导 AI
- 输出格式：纯文本，每行一个序号和文案

### 数据结构

#### `ReminderItemView`

```typescript
interface ReminderItemView {
  name: string;
  brand: string | null;
  quantity: number;
  unit: string;
  expiryDate: string | null;
  daysLeft: number | null;
  aiTip?: string | null; // AI 生成的幽默提示
}
```

### 格式化函数

- `formatExpiryLine()` - 文本版本（过期/临期物品）
- `formatStockLine()` - 文本版本（库存不足物品）
- `formatExpiryHtml()` - HTML 版本（过期/临期物品）
- `formatStockHtml()` - HTML 版本（库存不足物品）

所有格式化函数都会检查 `item.aiTip`，如果存在则添加到输出中。

### 常见问题

**Q: 为什么邮件中没有 AI 文案？**

A: 可能的原因：

1. AI 功能未启用或配置错误
2. API Key 无效或余额不足
3. AI 请求超时或失败（会降级到默认文案）
4. 检查控制台是否有错误日志

**Q: AI 文案不够幽默怎么办？**

A:

1. 调高 AI 温度参数（0.7-1.0）
2. 更换 AI 模型（如 gpt-4o）
3. 修改提示词中的示例和要求

**Q: 可以自定义文案风格吗？**

A: 可以！修改 `generateAiTipsForItems()` 函数中的 `prompt` 变量，调整：

- 角色定位
- 语言风格
- 示例文案
- 输出要求

**Q: AI 文案生成会影响邮件发送速度吗？**

A: 不会！

- AI 请求在邮件发送前完成
- 批量并行处理（每批 10 个物品）
- 失败时自动降级，不阻塞邮件发送
- 通常 2-5 秒完成所有文案生成
