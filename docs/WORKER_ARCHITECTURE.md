## 🔄 工作流程

### 1. Cron 触发流程

```
时间到达（例如：每天 UTC 1:00）
    │
    ▼
Cloudflare 自动调用 Worker 的 scheduled() 函数
    │
    ▼
executeCronJob() 开始执行
    │
    ├─► 验证环境变量（CRON_SECRET, API_BASE_URL）
    │
    ├─► 构造 HTTP 请求
    │   ├─ URL: ${API_BASE_URL}/api/cron
    │   ├─ Method: POST
    │   ├─ Headers:
    │   │   ├─ Content-Type: application/json
    │   │   ├─ x-cron-secret: ${CRON_SECRET}
    │   │   └─ User-Agent: CloudflareWorker/1.0
    │   └─ Timeout: 25 秒
    │
    ├─► 发送请求到应用
    │
    ├─► 等待响应
    │
    ├─► 解析响应
    │   ├─ 成功：记录成功日志
    │   └─ 失败：记录错误日志
    │
    └─► 返回执行结果
```

### 2. 手动触发流程

```
用户发送 POST /trigger 请求
    │
    ├─► 验证 Authorization Header
    │   ├─ 格式：Bearer ${CRON_SECRET}
    │   ├─ 匹配：继续执行
    │   └─ 不匹配：返回 401
    │
    ├─► 调用 executeCronJob()
    │   └─► （同上 Cron 触发流程）
    │
    └─► 返回 JSON 响应
        ├─ success: true/false
        ├─ data: 执行结果
        └─ duration: 执行时长（毫秒）
```

### 3. 应用端处理流程

```
接收到 Worker 的请求
    │
    ├─► 验证 x-cron-secret Header
    │   ├─ 匹配：继续执行
    │   └─ 不匹配：返回 403
    │
    ├─► 检查 Cron 是否启用
    │   ├─ 启用：继续执行
    │   └─ 禁用：返回跳过状态
    │
    ├─► 查询数据库
    │   ├─ 获取启用邮件通知的用户
    │   ├─ 获取需要提醒的物品
    │   │   ├─ 已过期物品
    │   │   ├─ 即将过期物品
    │   │   └─ 库存不足物品
    │   └─ 获取邮件配置
    │
    ├─► 生成邮件内容
    │   ├─ 调用 AI 生成幽默文案（可选）
    │   ├─ 生成邮件标题
    │   ├─ 生成纯文本内容
    │   └─ 生成 HTML 内容
    │
    ├─► 发送邮件
    │   ├─ 遍历每个用户
    │   ├─ 调用邮件服务（Resend/SMTP）
    │   ├─ 记录发送结果
    │   └─ 创建系统通知
    │
    ├─► 记录 Cron 日志
    │   ├─ 检查的物品数
    │   ├─ 发送的通知数
    │   ├─ 执行状态
    │   └─ 错误信息（如有）
    │
    └─► 返回执行结果
```

## 🔐 安全机制

### 1. 认证流程

```
┌─────────────┐
│   Worker    │
└──────┬──────┘
       │
       │ 1. 读取 env.CRON_SECRET
       │
       ▼
┌─────────────────────────────┐
│  构造请求                    │
│  Header: x-cron-secret       │
└──────┬──────────────────────┘
       │
       │ 2. 发送请求
       │
       ▼
┌─────────────┐
│   应用      │
└──────┬──────┘
       │
       │ 3. 读取 env.CRON_SECRET
       │
       ▼
┌─────────────────────────────┐
│  验证密钥                    │
│  request.headers.get()       │
│  === env.CRON_SECRET         │
└──────┬──────────────────────┘
       │
       ├─► 匹配：继续执行
       │
       └─► 不匹配：返回 403
```

### 2. 安全特性

| 特性           | 说明                                  |
| -------------- | ------------------------------------- |
| **密钥加密**   | `CRON_SECRET` 使用 Encrypted 类型存储 |
| **HTTPS 强制** | 所有请求必须使用 HTTPS                |
| **超时保护**   | 请求超时时间 25 秒                    |
| **错误隔离**   | Worker 错误不影响应用                 |
| **日志记录**   | 所有请求都有详细日志                  |

## 📊 数据流

### 请求数据流

```
Worker                    应用                    外部服务
  │                        │                        │
  ├─► POST /api/cron ─────►│                        │
  │   x-cron-secret        │                        │
  │                        │                        │
  │                        ├─► 查询 D1 数据库       │
  │                        │   - 用户列表           │
  │                        │   - 物品列表           │
  │                        │   - 系统配置           │
  │                        │                        │
  │                        ├─► 调用 AI API ────────►│
  │                        │   生成幽默文案         │
  │                        │◄─────────────────────┤
  │                        │   返回文案             │
  │                        │                        │
  │                        ├─► 调用邮件 API ───────►│
  │                        │   发送邮件             │
  │                        │◄─────────────────────┤
  │                        │   返回结果             │
  │                        │                        │
  │                        ├─► 写入 D1 数据库       │
  │                        │   - Cron 日志          │
  │                        │   - 系统通知           │
  │                        │                        │
  │◄─── 返回结果 ──────────┤                        │
  │   { success, data }    │                        │
```

```typescript
// Worker 返回给调用者

  };
  error?: string;
  duration: number;  // 毫秒
}
```

## ⚡ 性能优化

### 1. 边缘计算

```
用户位置          最近的边缘节点        应用服务器
   │                   │                   │
   │                   │                   │
   ├─► 请求 ──────────►│                   │
   │   (低延迟)        │                   │
   │                   │                   │
   │                   ├─► 执行 Worker     │
   │                   │   (边缘计算)      │
   │                   │                   │
   │                   ├─► API 请求 ──────►│
   │                   │   (仅一次)        │
   │                   │◄──────────────────┤
   │                   │   返回结果         │
   │                   │                   │
   │◄─── 响应 ─────────┤                   │
   │   (低延迟)        │                   │
```

### 2. 冷启动优化

| 方案                  | 冷启动时间 | 说明            |
| --------------------- | ---------- | --------------- |
| **Cloudflare Worker** | <1ms       | V8 Isolate 技术 |
| **传统 Serverless**   | 100-1000ms | 容器启动        |
| **GitHub Actions**    | 10-30s     | 虚拟机启动      |

### 3. 并发处理

```
Cron 触发（UTC 1:00）
    │
    ├─► Worker 实例 1 ──► 处理请求
    ├─► Worker 实例 2 ──► 处理请求
    ├─► Worker 实例 3 ──► 处理请求
    └─► ...

注：Cloudflare 自动扩展，无需配置
```

## 🚨 错误处理

### 1. 错误类型

```
┌─────────────────────────────────────┐
│           Worker 错误                │
├─────────────────────────────────────┤
│  - 环境变量未配置                    │
│  - 网络请求失败                      │
│  - 请求超时                          │
│  - 响应解析失败                      │
└─────────────────────────────────────┘
                │
                ▼
        记录日志 + 返回错误响应

┌─────────────────────────────────────┐
│           应用错误                   │
├─────────────────────────────────────┤
│  - 认证失败                          │
│  - 数据库查询失败                    │
│  - AI 调用失败                       │
│  - 邮件发送失败                      │
└─────────────────────────────────────┘
                │
                ▼
        记录日志 + 返回错误响应
```

### 2. 错误恢复

```
执行失败
    │
    ├─► Worker 记录错误日志
    │
    ├─► 应用记录 Cron 日志
    │   └─ status: 'failed'
    │
    ├─► 等待下次 Cron 触发
    │   └─ 自动重试
    │
    └─► 或手动触发
        └─ POST /trigger
```
