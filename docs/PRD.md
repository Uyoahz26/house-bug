> 版本：v1.0.0 | 更新日期：2026-04-15

---

## 一、产品概述

**产品名称**：HomeBug 小小虫  
**产品定位**：家庭物资库存管理 Web App，帮助用户管理日常生活物资（洗发水、食品、日用品等），支持拍照 OCR 录入、保质期追踪、过期提醒。  
**目标用户**：个人 / 家庭用户  
**核心价值**：减少物资浪费、避免重复采购、提醒过期商品

## 二、核心功能模块

### 2.1 用户认证

| 功能               | 描述                                         |
| ------------------ | -------------------------------------------- |
| 首次登录初始化     | 首个登录系统的账号自动创建并赋予 admin 权限  |
| 用户登录           | JWT 鉴权，支持 Remember Me（7天 token）      |
| 登出               | 清除本地 token                               |
| 用户管理（管理员） | 管理员可添加家庭成员账号、禁用账号、重置密码 |
| 个人设置           | 修改昵称、密码、通知偏好                     |

### 2.2 物资录入（OCR 智能录入）

**核心流程**：

```
用户上传图片 / 拍照
    → 调用 OCR API 识别文字
        → 自动提取关键字段（商品名、生产日期、保质期、厂家）
            → 预填表单
                → 用户确认并补充（位置、数量、备注等）
                    → 保存到 D1 数据库
```

**录入字段（完整表单）**：

| 字段          | 类型      | 必填 | 来源                                 |
| ------------- | --------- | ---- | ------------------------------------ |
| 商品名称      | 文本      | ✅   | OCR + 用户编辑                       |
| 商品图片      | 图片      | ❌   | 用户上传                             |
| 品牌 / 厂家   | 文本      | ❌   | OCR + 用户编辑                       |
| 分类          | 下拉选择  | ✅   | 用户选择（食品/日化/清洁/医疗/其他） |
| 规格/净含量   | 文本      | ❌   | OCR + 用户编辑（如 500ml、250g）     |
| 数量          | 数字      | ✅   | 用户输入，默认 1                     |
| 单位          | 文本      | ❌   | 用户输入（瓶、袋、盒等）             |
| 生产日期      | 日期      | ❌   | OCR 自动识别                         |
| 保质期        | 文本/数字 | ❌   | OCR 自动识别（如 24个月）            |
| 到期日期      | 日期      | ❌   | 自动计算（生产日期 + 保质期）        |
| 存放位置      | 文本/标签 | ❌   | 用户输入（冰箱/浴室/储物柜等）       |
| 采购日期      | 日期      | ❌   | 默认今天                             |
| 采购价格      | 数字      | ❌   | 用户输入                             |
| 采购渠道      | 文本      | ❌   | 用户输入（超市/淘宝/京东）           |
| 条形码/二维码 | 文本      | ❌   | 扫码识别（预留）                     |
| 备注          | 长文本    | ❌   | 用户输入                             |
| 标签 Tags     | 多选标签  | ❌   | 用户自由添加                         |

### 2.3 库存管理

| 功能      | 描述                                      |
| --------- | ----------------------------------------- |
| 物资列表  | 卡片 + 列表两种视图，支持搜索、过滤、排序 |
| 分类浏览  | 按分类（食品/日化/清洁/医疗/其他）筛选    |
| 快速编辑  | 点击直接修改数量（+/-）、删除             |
| 批量操作  | 批量删除、批量修改分类                    |
| 导入/导出 | 支持 CSV 导入导出                         |
| 物资详情  | 查看完整信息及历史记录                    |

### 2.4 保质期管理 & 提醒

| 状态     | 条件           | UI 标识     |
| -------- | -------------- | ----------- |
| 正常     | 距过期 > 30 天 | 绿色        |
| 即将过期 | 距过期 ≤ 30 天 | 橙色 / 警告 |
| 已过期   | 已超过到期日   | 红色        |
| 无保质期 | 未填写         | 灰色        |

**提醒机制**：

- Cloudflare Cron Trigger 每日运行，检查保质期
- 提醒推送（浏览器通知 / 邮件，可配置）
- Dashboard 过期预警卡片

### 2.5 数据统计 Dashboard

| 统计项     | 描述                     |
| ---------- | ------------------------ |
| 总物资数量 | 当前库存总品类数         |
| 即将过期   | 30天内过期数量           |
| 已过期     | 已过期物资数量           |
| 本月采购   | 本月新增物资数量         |
| 按分类分布 | 饼图或柱状图显示各类数量 |
| 最近添加   | 最近5条新增记录          |

### 2.6 设置中心

| 设置项    | 描述                                              |
| --------- | ------------------------------------------------- |
| 账户设置  | 昵称、邮箱、密码修改                              |
| 存储配置  | 图床来源（自定义 URL / 腾讯 COS / Cloudflare R2） |
| OCR 配置  | 默认 Tesseract.js，可切换 PaddleOCR/云 OCR        |
| 通知设置  | 过期提醒提前天数（默认7天、15天、30天）           |
| Cron 设置 | 自定义定时任务执行时间                            |
| 分类管理  | 自定义增删物资分类                                |
| 位置管理  | 自建常用存放位置标签                              |
| 数据管理  | 导入/导出/清空数据                                |

系统级配置（general/storage/ocr/email/cron）统一存储在 system_config 表中，由管理员在设置页修改。

## 三、用户旅程（主流程）

### 3.1 首次使用

```
登录（若系统无用户则自动创建首个管理员）
    → 引导页（功能介绍）
    → Dashboard（空状态引导添加物资）
```

### 3.2 添加物资（OCR 流程）

```
点击「+ 添加」→ 选择「拍照/上传图片」
    → 调用摄像头 / 本地相册
        → 图片预览确认
            → 调用 OCR API（加载动画）
                → 识别结果展示（高亮已识别字段）
                    → 用户确认/修改表单字段
                        → 提交保存
                            → 成功反馈 → 返回列表
```

### 3.3 手动添加

```
点击「+ 添加」→ 选择「手动填写」→ 填写表单 → 提交保存
```

### 3.4 过期提醒处理

```
Cron 每日执行 → 查找即将过期/已过期物资
    → 生成通知记录
        → 用户登录后查看通知 → 处理（标记废弃/已消耗）
```

```

/login              → 登录页

```

## 六、OCR 方案

**推荐选项**（按优先级）：

1. **Tesseract.js（前端默认）** — 开源免费，无需后端
2. **PaddleOCR（自托管，可选）** — 开源方案，中文识别更强
3. **Cloudflare AI（Workers AI，可选）** — 云端推理，接入成本低
4. **腾讯云 OCR API（可选）** — 商业 OCR，中文识别效果好
5. **百度 OCR API（可选）** — 商业 OCR，通用文字识别

**建议**：默认使用 Tesseract.js（开源优先），在设置页支持切换到 PaddleOCR 或云 OCR。

**设计理念**：

- 寓意：小小的虫子守护着家里的每一样东西
