# HomeBug 库存管理 Skill - 更新日志

## [1.1.0] - 2026-04-24

### ✨ 新增功能

#### 1. 查询已过期物品

- 支持查询所有已过期的物品
- 显示过期日期和存放位置
- 触发词：`有哪些东西已经过期了`、`过期的物品`、`已经过期`

**示例**：

```
你：有哪些东西已经过期了
OpenClaw：发现 2 件已过期物品：
- 牛奶（蒙牛）：1 瓶，过期日期 2026-04-20，存放在冰箱
- 面包（桃李）：1 袋，过期日期 2026-04-22，存放在厨房
```

#### 2. 查询即将过期物品

- 支持查询未来 7 天内即将过期的物品
- 显示剩余天数和存放位置
- 按过期日期排序（最快过期的在前）
- 触发词：`哪些东西快过期了`、`即将过期`、`快到期了`

**示例**：

```
你：哪些东西快过期了
OpenClaw：发现 3 件物品将在 7 天内过期：
- 酸奶（伊利）：2 盒，还有 3 天过期，存放在冰箱
- 鸡蛋：8 个，还有 5 天过期，存放在冰箱
- 火腿肠：1 包，还有 6 天过期，存放在厨房
```

#### 3. 查询特定位置物品

- 支持查询特定位置的所有物品
- 自动显示过期信息（已过期或即将过期）
- 支持模糊匹配位置名称
- 触发词：`冰箱里有什么`、`厨房有哪些`、`XX位置的物品`

**示例**：

```
你：冰箱里有什么
OpenClaw：冰箱里有 5 件物品：
- 牛奶（蒙牛）：2 瓶
- 酸奶（伊利）：2 盒，还有 3 天过期
- 鸡蛋：8 个，还有 5 天过期
- 黄油：1 盒
- 啤酒：6 瓶
```

### 🔧 技术改进

#### API 增强

- 新增 `query_expired` 操作类型
- 新增 `query_expiring` 操作类型（支持自定义天数阈值）
- 新增 `query_location` 操作类型
- 优化 AI 解析逻辑，支持更多自然语言表达

#### 数据结构

```typescript
interface ParsedAction {
  type:
    | "decrease"
    | "increase"
    | "query"
    | "query_expired"
    | "query_expiring"
    | "query_location";
  itemName?: string;
  quantity: number;
  location?: string;
  daysThreshold?: number;
}
```

1. **定期检查**
   - 每周检查一次即将过期物品

2. **位置管理**
   - 合理设置物品存放位置

3. **过期提醒**
